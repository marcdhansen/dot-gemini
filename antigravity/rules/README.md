# Global Rules and Standards

This directory contains universal rules and standards that govern all agent behavior across workspaces.

## Core Rules

The primary rules are maintained in the global Standard Operating Procedures:

- **[Global Rules](../docs/sop/SOP.md)** - Complete Protocol
- **[Protocol Standards](../docs/sop/NOMENCLATURE.md)** - Universal terminology
- **[Compatibility Rules](../docs/sop/CROSS_COMPATIBILITY.md)** - Multi-IDE/agent standards

## Purpose

Global rules provide:

1. **Universal Behavior** - Consistent agent conduct across all contexts
2. **Safety Standards** - Protocols to ensure safe and reliable operations
3. **Quality Assurance** - Standards for maintaining high-quality work
4. **Coordination Protocols** - Rules for multi-agent collaboration

## Enforcement

Rules are enforced through:

1. **Orchestrator Skill** - Automated compliance checking
2. **Initialization Checks** - Mandatory validation before work
3. **Finalization Checks** - Comprehensive session completion checks
4. **Continuous Monitoring** - Ongoing compliance during operations

## Evolution

Rules evolve through:

1. **Usage Feedback** - Learning from real-world application
2. **Incident Analysis** - Improving based on issues and failures
3. **Best Practice Integration** - Incorporating proven patterns
4. **Community Standards** - Adopting industry best practices
